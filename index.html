What is GitHub Pages?

GitHub Pages is a free static site hosting service from GitHub that turns the content of a repository into a live website. It is ideal for personal sites, project documentation, portfolios, and small marketing pages.

  • Free hosting directly from a GitHub repository.
  • Supports HTML, CSS, and JavaScript out of the box.
  • Built-in support for Jekyll for static site generation.
  • Custom domains and automatic HTTPS via Let's Encrypt.
  • Deploys straight from a branch (e.g. main) or a /docs folder.

Key Features

Zero Cost

Host your static site for free on GitHub's global infrastructure.

Git-Powered

Every git push to your Pages branch redeploys the site automatically.

Custom Domain

Point your own domain with a simple CNAME file.

HTTPS by Default

Sites are served over HTTPS with certificates managed for you.

How to Host a Site on GitHub Pages

  1. Create a new public repository on GitHub.
  2. Add your index.html, CSS, and JS files to the repo.
  3. Commit and git push your changes to the main branch.
  4. Open Settings → Pages in the repository.
  5. Under Build and deployment, choose Deploy from a branch.
  6. Select the main branch and / (root) folder, then Save.
  7. Wait a minute — your site is live at https://<user>.github.io/<repo>/.
